# Configuration File (`config.yaml`)

The behavior of Hermes can be customized through the `config.yaml` file located in the root of the workspace. This file is generated with default values by the `hermes init` command.

Values specified as options in the `hermes run` command will temporarily override the values in this file.

## Example of Default Configuration

```yaml
work_dir: /home/user/.hermes
ollama:
  api_url: http://localhost:11434/api/chat
  model: gpt-oss:20b
  timeout: 120
  temperature: 0.7
  max_retries: 3
search:
  query_count: 3
  min_search: 3
  max_search: 8
  max_results: 5
validation:
  min_validation: 1
  max_validation: 3
langfuse:
  enabled: false
  host: http://localhost:3000
  public_key:
  secret_key:
log:
  level: INFO
  retention: 7 days
```

## Parameter Details

### `work_dir`

-   **Description**: The absolute path to the workspace used by Hermes.
-   **Default**: `~/.hermes`

### `ollama`

Settings related to Ollama (the local LLM execution environment).

-   `api_url`: The API endpoint for Ollama.
-   `model`: The name of the default LLM model to use.
-   `timeout`: The timeout for API calls in seconds.
-   `temperature`: A parameter that controls the diversity of the model's responses. A lower value makes it more deterministic, while a higher value generates more diverse responses.
-   `max_retries`: The maximum number of retries when an API call fails.

### `search`

Settings related to web search.

-   `query_count`: The number of search queries to be generated by the LLM in the first step.
-   `min_search`: The minimum number of information sources to collect. If it falls below this, additional searches will be attempted.
-   `max_search`: The maximum number of information sources to collect.
-   `max_results`: The maximum number of search results to retrieve from SearxNG per search query.

### `validation`

Settings related to the loop for validating the quality of the generated report.

-   `min_validation`: The minimum number of validation loops to finalize the report.
-   `max_validation`: The maximum number of validation loops. If this is exceeded, the loop will terminate even if improvements are found.

### `langfuse`

Settings related to recording execution traces using [Langfuse](https://langfuse.com/).

-   `enabled`: Whether to enable trace recording to Langfuse.
-   `host`: The host URL of the Langfuse server.
-   `public_key`: The public key of the Langfuse project.
-   `secret_key`: The secret key of the Langfuse project.

### `log`

Settings related to logging.

-   `level`: The log level (`DEBUG`, `INFO`, `WARNING`, `ERROR`).
-   `retention`: The period for which log files are retained.
