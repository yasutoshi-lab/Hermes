# `hermes task` コマンド

`hermes task` コマンドは、リサーチタスクの管理（追加、一覧表示、削除）を行うために使用されます。

## 機能

このコマンドは、`hermes run` で後から実行するためのリサーチプロンプトをタスクとして永続化します。

-   **タスク追加**: 新しいリサーチプロンプトをタスクとして登録します。タスクには一意のIDが割り当てられます。
-   **タスク一覧**: 現在登録されているすべてのタスクを、ID、ステータス、プロンプト、作成日時と共にテーブル形式で表示します。
-   **タスク削除**: 指定されたタスクIDを持つタスクを削除します。

## オプション

| オプション | 説明 |
| :--- | :--- |
| `--add "PROMPT"` | 指定されたプロンプトで新しいタスクを作成します。 |
| `--list` | 登録されているすべてのタスクを一覧表示します。 |
| `--remove "TASK_ID"` | 指定されたタスクIDのタスクを削除します。 |
| `--work-dir PATH` | ワークディレクトリのパスを指定します。デフォルトは `~/.hermes` です。 |

## 実装詳細

-   **CLIフレームワーク**: `click`
-   **サービス**: `hermes_cli.services.task_service.TaskService`
-   **メインロジック**:
    1.  `TaskService` を初期化します。このサービスがタスクの永続化（ファイルの読み書き）を担当します。
    2.  `--add` オプションが指定された場合:
        -   `TaskService.create_task()` を呼び出し、プロンプトを内容とする新しいタスクファイルを作成します。
        -   作成されたタスクのIDとプロンプトを表示します。
    3.  `--list` オプションが指定された場合:
        -   `TaskService.list_tasks()` を呼び出し、ワークディレクトリ内のすべてのタスクファイルを読み込みます。
        -   `rich.table.Table` を使用して、タスクの一覧を整形されたテーブルで表示します。
    4.  `--remove` オプションが指定された場合:
        -   `TaskService.delete_task()` を呼び出し、指定されたIDに対応するタスクファイルを削除します。
    5.  いずれのオプションも指定されなかった場合、使用方法に関するメッセージを表示します。
